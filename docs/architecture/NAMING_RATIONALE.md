# Naming Rationale

This project favors short, memorable one-word names with technical expansions.

## Candidate set

1. haven
- Expansion: Hypervisor for Asymmetric Virtualization ENforcement and isolation.
- Meaning: A protected place; strong metaphor for partition safety boundaries.

2. spar
- Expansion: Static Partition hypervisor for Asymmetric multiprocessing in Real-time systems.
- Meaning: Short and terminal-friendly; emphasizes architecture directly.

3. shade
- Expansion: Static Hypervisor for Asymmetric Domain Enforcement.
- Meaning: Protection and shielding metaphor for interference prevention.

4. prism
- Expansion: Partition Runtime Isolation for Secure Multiprocessing.
- Meaning: Separation into distinct bands without mixing.

5. moat
- Expansion: Multicore OS Asymmetric Trustzone enforcement.
- Meaning: Physical barrier metaphor for spatial defense.

6. apex
- Expansion: Asymmetric Partition EXecutive.
- Meaning: Professional and concise aerospace-style vocabulary.

## Selected repository name

Official name: haven

Selection rationale:
- Strong safety metaphor aligned with thesis goals.
- Easy pronunciation and retention for academic references.
- Flexible for long-term evolution beyond thesis scope.
