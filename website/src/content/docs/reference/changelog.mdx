## [0.6.2] - 2026-06-01

### Added
- **Linker layout fix** (PR #42): `.boot.text` section now placed first at `0x80000000`;
  `el2_exception_table` aligns to `0x80000800` (2KiB). QEMU boot banner
  `"Haven hypervisor starting"` confirmed in UART output for the first time.
- **EL2 exception entry hardening** (PR #43): `entry.S` exception handlers hardened;
  `el2_exceptions.c` fault injection tests validated all 8 scenarios.
- **PSCI CPU_ON error-path rejection** (PR #44, T7): `hv_arch_psci_cpu_on()` returns
  `HV_EACCES` when a partition attempts to wake a non-owned secondary CPU.
- **FreeRTOS budget/period fidelity** (PR #44, T8): FreeRTOS task wakeups remain within
  one scheduler tick of configured period under cross-partition CPU stress.
- **CI toolchain version pins** (PR #47): top-level `env:` block in `ci.yml` pins
  `QEMU_VERSION`, `GCC_AARCH64_VERSION`, `CPPCHECK_VERSION`, `CLANG_VERSION`.
- **Coq advisory CI job** (PR #47): `coq-proofs` job runs `coqc` on
  `verification/coq/*.v` on every PR (continue-on-error: true; M-P4-1 gate).
- **Evidence analysis tooling** (PR #49): `tools/analysis/latency_analyzer.py` gains
  p99.9 estimate column; all three analysis scripts verified against live benchmark data.
- **DESCRIPTION.md Parts 12-13** (PR #50): CI/CD automation lifecycle (multi-board
  matrix, nightly split, artifact signing, toolchain refresh) and documentation
  living lifecycle (quarterly review, Doxygen, diagram refresh triggers).

## [0.6.0] - 2026-05-21

### Added
- **Secondary CPU bring-up**: `hv_arch_psci_cpu_on()` in `arch/arm64/boot.S` (PSCI CPU_ON SMC64 via HVC conduit). Partition B (RTOS, CPU 2) is now woken by PSCI before the ERET to Partition A. `partition_b_secondary_cpu_init()` enables stage-2 for VMID 2 and ERETSs into Partition B's IPA at `0x60000000`.
- **FreeRTOS context hardening**: `hv_freertos_save_context` / `hv_freertos_restore_context` now enforce task state-machine transitions (save requires RUNNING; restore requires READY/RUNNING). Weak-symbol arch hooks `hv_arch_context_save` / `hv_arch_context_restore` allow ARM64 overrides.
- `hv_freertos_task_block` / `hv_freertos_task_unblock` with priority inversion detection.
- 6 secondary CPU isolation integration tests (`tests/integration/test_secondary_cpu_isolation.c`).
- 4 new FreeRTOS hardening unit test groups (46 assertions total).
- CI benchmark regression gate with 10% threshold in `.github/workflows/nightly.yml`.
- EL2 virtual IRQ injection: `VBAR_EL2` wired + GICv3 LR programming.
- Partition B RTOS guest stub (`tests/demos/guest_b_entry.S`) with 3 RTOS task loops, deliberate stage-2 DABT, and WFI halt.
- SMMU integration test expanded to 8 fault-injection scenarios.

### Changed
- `hv_freertos_allocate_shared_region`: zero-size regions now rejected (`HV_EINVAL`) in addition to non-page-aligned.
- `scripts/qemu/qemu-smoke.sh`: removed stale Phase 3 placeholder comment; both partition A and B UART markers now report success.

### Fixed
- SMMU `reset_partition` memory leak in 8-scenario test expansion.
- `hv_budget_set` documentation corrected: `budget_ns == period_ns` is permitted (full-period monopoly); only zero and `budget_ns > period_ns` are rejected.
