## Release 1 (Months 1–3) - Foundation

- Benchmark and evidence packaging baseline. ✓
- Initial i.MX95 bring-up runbook and evidence templates. ✓

Mandatory checks:
- PR CI and nightly CI pass (gcc + clang). ✓
- Integration flow test passes. ✓
- Config validation passes. ✓

## Release 2 (Months 4–6) - Isolation Mechanisms

**Theme**: Core isolation mechanism maturity.

Deliverables:
- ARM64 architecture layer: EL2 exception vectors, context save/restore, stage-2 page tables. ✓
- GICv3 driver wired to IRQ ownership policy; SMMUv3 driver wired to DMA policy. ✓
- Formal Coq proofs for spatial isolation invariant, stage-2 policy, and budget scheduler. ✓
- 8 fault-injection scenarios (F1–F8) verified DENY via `test_fault_injection.c`. ✓
- Full benchmark suite: isolation latency, stage-2 fault, SMMU policy, temporal isolation. ✓
- ARM64 cross-compile CI gate and nightly Coq proof verification. ✓

Mandatory checks:
- All Release 1 checks still pass. ✓
- `test_isolation_negative` catches all denied accesses. ✓
- ARM64 binary (`build/haven.elf`) produced and verified by CI. ✓
- Coq proofs check under Coq 8.18. ✓

## Release 3 (Months 7–9) - Platform Validation

**Theme**: Platform validation depth and mixed-environment verification.

Deliverables:
- QEMU end-to-end boot with two-partition demo (UART evidence). ✓
- Secondary CPU bring-up: PSCI CPU_ON wires CPU 2 → Partition B. ✓ **(v0.6.0)**
- FreeRTOS context-switch hardening: state-machine validation + arch hooks. ✓ **(v0.6.0)**
- Full i.MX95 validation campaign with repeatable evidence capture.
- Cross-OS virtual validation (macOS / Linux / Windows).
- Secondary heterogeneous board baseline (i.MX8QM-MEK or equivalent).
- Linux guest kernel module (`drivers/linux/haven_driver.c`) and userspace CLI. ✓

Mandatory checks:
- All Release 2 checks still pass. ✓
- i.MX95 UART logs captured and archived.
- Latency measurements within declared bounds.

Mandatory checks:
